# Minecraft RAG 
---

## Overview

A RAG system built on top of the **naklecha/minecraft-question-answer-700k** dataset. The system retrieves relevant Minecraft knowledge from a ChromaDB vector database and generates answers using Hugging Face LLMs.

---

## Project Structure

```
rag_minecraft/
├── 01_data_and_indexing.ipynb   # Load dataset, generate embeddings, build ChromaDB
├── 02_experiments.ipynb         # Run 12 experiment combinations + top_k grid
```

## How to Run

### Step 1 — Index the data
Open and run `01_data_and_indexing.ipynb`.  
This downloads the dataset, generates embeddings, and creates two ChromaDB collections:
- `minecraft_minilm` — using `all-MiniLM-L6-v2`
- `minecraft_mpnet` — using `all-mpnet-base-v2`

-> Safe to re-run, skips indexing if collections already exist.

### Step 2 — Run experiments
Open and run `02_experiments.ipynb`.  
Runs all 12 combinations (2 embeddings × 2 LLMs × 3 prompts) + top_k grid [1, 3, 5].  
Saves results to `results/experiments.json`.

### Step 3 — Evaluate
Open and run `03_evaluation.ipynb`.  
Computes ROUGE-L, Hit@3, and latency on 100 held-out Q&A pairs.  
Generates 4 charts: ROUGE-L bar chart, latency comparison, heatmap, top_k line chart.

### Step 4 — Gradio demo
Open and run `04_gradio_app.ipynb`.  
The last cell launches the interface at **http://localhost:7860**.

## Models

| Component | Model | Notes |
|-----------|-------|-------|
| Embedding | `all-MiniLM-L6-v2` | 384 dims, fast, ~80MB |
| Embedding | `all-mpnet-base-v2` | 768 dims, slower, ~420MB |
| LLM | `Qwen/Qwen2.5-0.5B-Instruct` | Best quality in experiments |
| LLM | `HuggingFaceTB/SmolLM2-360M-Instruct` | 2× faster, lower quality |

## Prompt Templates

**Minimal** — bare context + question  
**Detailed** — expert persona, strict "use only context" instruction  
**Chain-of-Thought** — step-by-step reasoning prompt

## Key Results

| Config | ROUGE-L | Latency |
|--------|---------|---------|
| MiniLM + Qwen + detailed | **0.230** | 7.0s |
| MiniLM + Qwen + minimal | 0.222 | 7.0s |
| MPNet + Qwen + detailed | 0.226 | 7.2s |
| MiniLM + SmolLM + minimal | 0.209 | 3.3s |
| MiniLM + SmolLM + detailed | 0.042 | 3.3s |

**Best overall:** `MiniLM + Qwen + detailed + top_k=3` → ROUGE-L **0.234**

### Findings
1. **LLM choice dominates** — Qwen vs SmolLM gap (0.21 vs 0.09) far exceeds embedding model gap (0.007)
2. **SmolLM fails on complex prompts** — scores 0.04 on detailed/CoT; too small to follow instructions
3. **top_k=3 is optimal** — top_k=5 introduces noise and hurts ROUGE-L
4. **Speed-quality trade-off** — SmolLM is 2x faster but produces unusable answers on structured prompts

## Dataset

**naklecha/minecraft-question-answer-700k** on Hugging Face  
700,000 Minecraft Q&A pairs. This project uses the first **5,000 records**.  
- Documents indexed in ChromaDB: `answer` field  
- Eval queries: `question` field (100 held-out pairs)

## Gradio Interface

The demo in `04_gradio_app.ipynb` exposes:
- Free-text question input
- Dropdown: embedding model, LLM, prompt template
- Slider: top_k (1–5)
- Outputs: generated answer + 3 retrieved context blocks
- 3 pre-loaded example questions
